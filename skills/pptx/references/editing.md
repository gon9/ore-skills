# 既存プレゼンテーションの編集

既存の .pptx ファイルを編集するためのワークフロー。
PPTX は ZIP 形式の XML アーカイブなので、展開→XML編集→再パックで編集する。

## テンプレートベースのワークフロー

### 1. 既存ファイルの分析

```bash
# テキスト抽出
python -m markitdown presentation.pptx

# サムネイル画像の生成（レイアウト確認用）
python scripts/thumbnail.py presentation.pptx
```

`thumbnails.jpg` でレイアウトを確認し、`markitdown` の出力でテキスト内容を確認する。

### 2. スライドマッピングの計画

コンテンツのセクションごとに、テンプレートのどのスライドレイアウトを使うか決める。

**レイアウトのバリエーションを意識する:**
- 2カラムレイアウト
- 画像 + テキストの組み合わせ
- 全面画像 + テキストオーバーレイ
- 引用・コールアウトスライド
- セクション区切りスライド
- 統計・数字のコールアウト

**同じテキスト重視レイアウトの繰り返しは避ける。**

### 3. 展開

```bash
python scripts/unpack.py presentation.pptx unpacked/
```

展開されたディレクトリ構造:
```
unpacked/
├── [Content_Types].xml
├── _rels/
├── ppt/
│   ├── presentation.xml      # スライド順序の定義
│   ├── slides/
│   │   ├── slide1.xml         # 各スライドの内容
│   │   ├── slide2.xml
│   │   └── ...
│   ├── slideLayouts/          # レイアウト定義
│   ├── slideMasters/          # マスタースライド
│   ├── media/                 # 画像等
│   └── theme/                 # テーマ定義
└── docProps/
```

### 4. 構造変更

- **スライドの削除**: `presentation.xml` の `<p:sldIdLst>` から該当エントリを削除
- **スライドの複製**: `scripts/add_slide.py` でスライドを複製
- **スライドの並べ替え**: `<p:sldIdLst>` 内のエントリ順を変更

**構造変更はすべてコンテンツ編集の前に完了させること。**

### 5. コンテンツ編集

各 `slide{N}.xml` 内のテキストを編集する。

XML 内のテキスト構造:
```xml
<a:t>ここにテキスト</a:t>
```

**編集時の注意:**
- XML のタグ構造を壊さない
- スマートクォート（`'` `"` `'` `"`）はそのまま保持する
- フォーマット属性（`<a:rPr>`）は変更しない（テキスト内容のみ変更）

### 6. 再パック

```bash
python scripts/pack.py unpacked/ output.pptx
```

### 7. QA

SKILL.md の QA セクションに従って確認する。

## スクリプト一覧

| スクリプト | 用途 |
|-----------|------|
| `unpack.py` | PPTX を展開し XML を整形表示 |
| `pack.py` | 展開ディレクトリから PPTX を再構築 |

| `thumbnail.py` | スライドのサムネイルグリッド生成 |

## よくある落とし穴

### テンプレート編集時

- **XML タグを壊さない** — テキスト（`<a:t>` の中身）だけを変更する
- **名前空間を消さない** — XML の名前空間宣言は必須
- **メディアファイルを消さない** — 参照が壊れる

### 複数スライド編集時

- **構造変更を先にやる** — スライドの追加・削除・並べ替えを先に完了してから、テキスト編集に入る
- **スライド番号のズレに注意** — スライドを削除・追加すると番号がずれる

### スマートクォート

- PPTX 内では `'` `'` `"` `"` が使われることがある
- これらは XML エンティティとして正しくエスケープする必要がある
