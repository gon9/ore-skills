# Prompt Patterns

効果的なプロンプトを通じて、エージェントから高品質な回答とコードを引き出すための型です。

## Last Updated: 2026-03-29

## 現在のベストプラクティス

### 1. ペルソナと制約 (Role & Constraints)
- **Role**: 単なる「AIアシスタント」ではなく、「10年の経験を持つシニアPythonエンジニア」のような明確な役割を与える。
- **制約**: どういう手順を踏み、どのフォーマットで出力するかをリストで明記する。

### 2. 思考プロセス (Chain-of-Thought / Reflection)
- エージェントに「ステップバイステップで考えること」を指示する。
- 複雑なロジックをコーディングする前に、まずは重要な変数の状態変化やアルゴリズムをテキストで説明させ、最後にコードを出力させる。

### 3. 自己防衛 (Defensive Prompting)
- ツールが失敗した場合や、答えがわからない場合に「適当に推測する」ことを防止する。
- 万が一失敗した場合は「ユーザーに質問すること」や「わかる範囲内でのみ回答すること」を明示する。

### 4. Few-Shot (具体例の提示)
- システム構成やフォーマットの指示において、抽象的な説明よりも、期待する理想的な出力例を1〜2つ（Few-Shot Examples）載せる方が強力に機能する。

## 変化の兆候・注目トピック
- 自動的なプロンプト最適化機能の登場（人間が書いた簡潔な要件から、LLM自身が最適なシステムプロンプトに変換して実行する仕組み）。
