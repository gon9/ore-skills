# Workflow Patterns

AIエージェントと効率的かつ正確に協働するための作業プロセスのパターンです。

## Last Updated: 2026-03-29

## 現在のベストプラクティス

### 1. Plan-First ワークフロー
- すぐにコードを書かせない。
- 複雑なタスクでは、まずエージェントに現在のコードベースの調査と「実装計画（Implementation Plan）」の作成を指示する。
- 人間がその計画をレビューし、承認（Approve）してから実際のコーディング（Execution）に移らせる。

### 2. Atomic Tasks (極小タスクの原則)
- タスクは10〜15分程度で完了する最小単位に分割する。
- 認証設定、DBスキーマ変更、UI実装を1つの指示にまとめない。
- コンテキストが混ざる（Context Rot）とハルシネーションの確率が跳ね上がるため。

### 3. エラー時の学習サイクル
- エージェントが間違った書き方をした場合、コードを直すだけで終わらせない。
- 「なぜ間違えたか」を分析し、再発防止のルールを `CLAUDE.md` や `AGENTS.md` に追記して、エージェントを"教育"する。

### 4. レビュアーとしての人間
- 人間とエージェントは「ペアプログラミング」のパートナー。
- エージェントが出力したコードは、人間の同僚のPRをレビューするのと同じ厳格さで査読する。

## 変化の兆候・注目トピック
- エージェント自体が計画（Plan）と実行（Execute）のモードを自律的に切り替える仕組みの実装。

## ore-skills への示唆
- 💡 **既存スキル**: `spec` スキルは仕様書の構成をチェックし、実装前に「プラン」の妥当性を保つための重要なピース。
- 💡 **スキル候補**: タスク分割アシスタント（大きな機能要件を与えると、Atomic Task のリストに分解し Markdown で出力する）。
