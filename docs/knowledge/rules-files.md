# Rules & Instructions Files

AIエージェントにプロジェクト特有の規約や技術スタックを教えるための全体ルールの記述方法です。
`CLAUDE.md`, `AGENTS.md`, `.cursorrules` などが該当します。

## Last Updated: 2026-03-29

## 現在のベストプラクティス

### 1. 階層的・モジュール化 (Progressive Disclosure)
- ルールを一つの巨大なファイルに詰め込まない。
- ルートの `CLAUDE.md` や `AGENTS.md` には、アーキテクチャの概要や全体規約のみを記述する。
- APIの仕様やテストの書き方など、特定の領域に関する詳細なルールは `.claude/rules/*.md` のような別ファイルに分割して参照させる。

### 2. 記述のコツ
- **命令形を使う**: 「このプロジェクトはReactを使っています」ではなく「Reactを使用して実装してください」と書く。
- **否定の制約を明記する**: 「〜しないこと」「〜は禁止」という制約は、エージェントの暴走を防ぐのに非常に効果的。
- **テスト可能なルール**: 「プロフェッショナルなコードを書く」のような曖昧な表現を避け、「型推論に頼らず明示的に型定義をする」のように真偽が判定できる書き方にする。

### 3. 一元管理と共有
- 複数のAIツール（Cursor, Windsurf, Claude Code等）を使用する場合、汎用的な `AGENTS.md` を一つ作り、各ツール専用のファイル（`.cursorrules` など）から読み込む（シンボリックリンク等）運用が主流。

## 変化の兆候・注目トピック
- ベンダー非依存の `AGENTS.md` 標準の普及。

## ore-skills への示唆
