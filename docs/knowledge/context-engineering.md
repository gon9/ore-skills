# Context Engineering

コーディングエージェントが正確かつ予測可能なコードを出力するには、モデルに与える情報（コンテキスト）を厳密に管理する「コンテキストエンジニアリング」が不可欠です。

## Last Updated: 2026-03-29

## 現在のベストプラクティス

### 1. 関連情報の絞り込み (Relevance)
- **ノイズの排除**: タスクに無関係なファイルや過去の長大なログをコンテキストに含めない。
- **Just-in-Time Context**: 必要なタイミングで必要な情報だけをツールキット経由で取得させる（ディレクトリ検索、grep検索など）。

### 2. コンテキストの劣化防止 (Context Rot)
- **スレッドの使い分け**: 話題やタスクが変わる（例：フロントエンドの実装からバックエンドのバグ調査に移る）場合は、必ずチャットスレッドを新しくする。
- **要約の活用**: 長時間続くタスクでは、中間の試行錯誤を要約し、新しいスレッドの初期プロンプトとして引き継ぐ。

### 3. Progressive Disclosure (段階的開示)
- 情報を一度にすべて与えるのではなく、エージェントが必要だと判断したタイミングで詳細を開示する階層構造を作る。
- 例：`ore-skills` の Skill 定義における `SKILL.md` (概要) → `references/*.md` (詳細) の構造。

## 変化の兆候・注目トピック
- 自動的なコンテキスト最適化（エージェント自身が不要なコンテキストを破棄・圧縮する仕組みの普及）。
