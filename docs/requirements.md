# 要件定義書

## 1. プロジェクトの目的
AIエージェント開発を加速させるための「再利用可能なスキル（ツール）群」を構築・管理する。
特に「Spec（仕様書）開発」と「情報収集（YouTube要約等）」を初期の主要機能とする。

## 2. 非機能要件
- **管理性**: 複数のプロジェクトから容易に参照・利用できること。依存関係の競合を避けること。
- **保守性**: コードは適切にモジュール化され、テスト（pytest）とドキュメント（docstring）が整備されていること。
- **品質**: 型ヒント（Type Hints）を完備し、Ruffによる静的解析を通すこと。
- **拡張性**: 新しいスキルを容易に追加できるディレクトリ構成であること。
- **インターフェース**: 
    - Pythonライブラリとして直接import可能であること。
    - AIエージェント（Windsurf, Claude Desktop等）から利用可能な **MCP (Model Context Protocol)** サーバーとして動作すること。

## 3. 機能要件

### 3.1. Spec開発支援スキル (packages/spec-skills)
- **要件定義生成**: ユーザーの曖昧な要望から、構造化された要件定義書案を生成する。
- **仕様書チェック**: 既存の仕様書（Markdown）を読み込み、矛盾点や不足項目を指摘する。
- **ドキュメント管理**: プロジェクトのドキュメント構造を提案・生成する。

### 3.2. メディア処理スキル (packages/media-skills)
- **YouTube文字起こし**: 指定されたYouTube動画のURLから、字幕（Transcript）を取得する。
- **動画要約**: 文字起こしテキストを要約し、要点を抽出する。
- **Webコンテンツ取得**: 指定URLのWebページコンテンツを取得し、LLMが読みやすい形式（Markdown等）に変換する。

### 3.3. 共通基盤 (packages/common)
- **ロギング**: 統一されたログ出力設定。
- **設定管理**: 環境変数（.env）や設定ファイルの読み込み機能。
- **MCPユーティリティ**: MCPサーバー実装のためのヘルパー関数群。

## 4. 技術スタック
- **言語**: Python 3.12
- **パッケージ管理**: uv (Workspace機能を使用)
- **リンター/フォーマッター**: ruff
- **テスト**: pytest
- **AIインターフェース**: model-context-protocol (python-sdk)
